Date: 2025-12-11
Issue: Email links were displaying incorrect URLs
Status: FIXED

================================================================================
PROBLEM DESCRIPTION
================================================================================

The email button links were displaying:
https://wilsonmisor.github.io/wilson-portfolios/wilsonudomisor/wilsonudomisor@gmail.com

Instead of:
mailto:wilsonudomisor@gmail.com

Root Cause:
- Email links had both href="mailto:wilsonudomisor@gmail.com" AND data-link-path="links.email"
- JavaScript function applyLinkBindings() was reading from site-config.json
- The config had "email": "wilsonudomisor@gmail.com" (without mailto:)
- JavaScript was overriding the correct href with the config value
- Since the email address didn't start with "mailto:", it was treated as a relative URL

================================================================================
SOLUTION APPLIED
================================================================================

Removed the data-link-path="links.email" attribute from all email links.

The links now use only:
<a class="btn primary" href="mailto:wilsonudomisor@gmail.com">Email</a>

This allows the browser to handle the email link correctly without JavaScript interference.
